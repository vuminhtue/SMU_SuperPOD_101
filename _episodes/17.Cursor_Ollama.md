# Cursor
- Very good IDE to use with Jupyter Notebook
- It attaches propertary API

# Ollama
- Ollama is an open-source framework that enables users to run, create, and manage large language models (LLMs) locally on their computers and on HPC system
- How to use Ollama local LLM to run with Cursor with Jupyter Notebook file on SuperPOD?

# Step by step:

## NGROK
- Register a free account on https://ngrok.com/
- Register a free Authenticate token
- Download ngrok executable file for SuperPOD from here:
```
https://ngrok.com/download/linux?tab=download
```
- Transfer to SuperPOD and Extract the executable:
```
tar -xvzf ~/Downloads/ngrok-v3-stable-linux-amd64.tgz
```
- Open the executable with the previous Authenticate toke:
```
$ ./ngrok config add-authtoken abcdef
```

- Next pipe the ollama port to internet:
```
$ ./ngrok http 11434 --host-header="127.0.0.1:11434"
```

- The internet link with url will occur and copy that
**DONE This step**

## Cursor
- Back to Cursor, go to Cursor Setting: API Keys:
- Enable OpenAI API Key and use: **ollama** as the key
- Enable Overrider OpenAI Base URL and paste in the link from Ngrok
- In **Add Custom Model**, add **gpt-oss:20b, qwen3.5:9b, llama3.1:8b** (These are the models support tooling)
- Now you are ready to use Cursor with AI supported on SPOD
